# Розробка додатку для візуалізації вимірювань GPS

## Короткий опис роботи
Цей проект присвячений розробці веб-додатку для візуалізації та аналізу GPS-даних у реальному часі. Додаток отримує дані про положення супутників через протокол WebSocket, обробляє їх та розраховує координати об'єкта двома різними підходами.

**Мета роботи:**
Метою роботи є практичне вивчення принципів трилатерації та методів чисельної оптимізації. У проекті реалізовано:
* **Аналітичний метод**: пряме розв'язання системи лінійних рівнянь для швидкого отримання координат.
* **Чисельний метод (Градієнтний спуск)**: ітеративний пошук оптимальних координат шляхом мінімізації функції втрат (похибки відстані).

## Інструкції для запуску проекту

### Вимоги до середовища:
* **Docker**: для запуску серверної частини (емулятора).
* **Веб-браузер**: для відображення фронтенд-частини.

### Покроковий запуск:
1. **Запуск емулятора GPS**:
   Відкрийте термінал та виконайте команду для запуску Docker-контейнера:
   ```bash
   docker run --name gps-emulator -p 4001:4000 iperekrestov/university:gps-emulation-service
   ```
2.  **Запуск веб-додатку**:
    Відкрийте файл `index.html` безпосередньо у браузері. Додаткове встановлення залежностей (npm, maven тощо) не потрібне, оскільки додаток реалізований на чистому JavaScript.

## Результати роботи
Додаток відображає динамічний рух об'єктів на сітці:
* **Сині кола**: супутники, що передають сигнал.
* **Червоний маркер (ромб)**: положення об'єкта, розраховане аналітичним методом.
* **Зелений маркер (трикутник)**: положення об'єкта, розраховане чисельним методом.

## Висновок
Під час виконання лабораторної роботи було закріплено навички роботи з WebSocket для отримання даних у реальному часі та використання REST API для керування параметрами системи. Ключовим інсайтом став порівняльний аналіз методів розрахунку координат: аналітичний метод є оптимальним для швидкого отримання точних результатів за відсутності перешкод, тоді як чисельний метод демонструє вищу стійкість до похибок у реальних даних. Використання графічної візуалізації на базі Chart.js дозволило наочно оцінити динаміку руху об'єктів та ефективність обох алгоритмів трилатерації.
